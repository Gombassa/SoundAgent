Review the current conversation for insights worth persisting to CLAUDE.md files or memories.

## What to look for

1. **New patterns or conventions** discovered during this session that should be in a CLAUDE.md file (repo-level or workspace-level) so future sessions don't rediscover them.
2. **Gotchas or bugs encountered** that should be added to the Gotchas section of the relevant CLAUDE.md.
3. **User preferences or workflow feedback** that should be saved as memories (feedback type).
4. **Cross-repo patterns** that should be in the workspace CLAUDE.md or shared patterns memory.
5. **Stale information** in existing CLAUDE.md files or memories that should be updated based on what changed this session.

## What NOT to save

- Code changes, file paths, or architecture that can be derived by reading the current codebase
- Anything already captured in CLAUDE.md files or memories
- Ephemeral task details only relevant to this session

## Steps

1. Scan the conversation for candidates in each category above.
2. For each candidate, state what it is and where it should go (which file).
3. Ask the user for approval before writing anything.
4. Write the approved changes.
